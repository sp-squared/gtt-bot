    Here's a full picture of DIF from the vault:
    
    
    
    WHAT IS DIF?
    
    Deterministic Intent Folding (DIF) is a specialized machine learning approach developed at Merly. It is NOT a generic design principle and NOT a transformer/LLM -- it is Merly's specific proprietary technology.
    
    
    
    CORE PROPERTIES
    
      Determinism
      The same input always produces the same output. LLMs are stochastic -- the same prompt can produce different outputs. DIF does not have this problem.
    
      Speed
      DIF inference is up to 1,000,000x faster than LLMs. This is constant-time inference, not just optimization. Because it's so fast, it runs entirely on-device with no cloud dependency, no data leaving the machine.
    
      Local inference
      Runs fully locally due to the speed advantage.
    
    
    
    WHAT DIF ENABLES: Repository Lifetime Reasoning (RLR)
    
    The combination of determinism + speed enables RLR -- the ability to analyze the entire evolution of a software repository, not just its current snapshot.
    
    The vault uses a good analogy: a photograph of a moving train can't tell you which direction it's going. A 5-second video can. Temporal context reveals intentionality that point-in-time analysis misses.
    
    This is "temporal plane 2" -- reasoning about how a codebase became what it is, not just what it is now.
    
    Why LLMs can't replicate this: transformers scale quadratically with input length. DIF does not. Analyzing years of commit history requires constant-time inference at scale -- only DIF provides that.
    
    
    
    MENTOR
    
    Mentor is Merly's first product built on DIF. It analyzes repos for signs of vibe coding, runs entirely locally, and contributes to a global database cataloguing vibe-coded open source projects.
    
    What Mentor detects that snapshot analysis misses: wholesale file replacements, generic commit messages revealing no understanding, absence of iterative refinement, unrelated changes bundled in single commits because the AI generated them as a package.
    
    
    
    DIF'S ROLE IN THE AI STACK
    
    DIF is not a replacement for LLMs -- it's a guardrail/verifier. LLMs are powerful but stochastic and unreliable. DIF is the deterministic verification layer that sits under LLMs and makes their outputs trustworthy.
    
    The competitive landscape note is pointed: all major AI systems (Google, Amazon, Meta, Microsoft, Apple) are stochastic transformers. This is an architectural difference, not a gap they can close by scaling.
